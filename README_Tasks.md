# README_tasks.md

## Pattern.Adapter задачи

1. Нужно заряжать телефон от обычной европейской розетки, но телефон ожидает USB-интерфейс. Реализуй переходник между EuropeanSocket и IUsbCharger.

2. Есть старая библиотека оплаты, которая умеет только PayFromTerminal(), а новый код ожидает метод PayOnline(). Сделай адаптер, чтобы старая система работала в новом интерфейсе.

3. Есть внешний сервис доставки, который возвращает адрес в одном формате, а твоя программа работает с другим. Реализуй адаптер для преобразования данных.

4. Есть класс принтера, который умеет только PrintText(), но твой редактор вызывает PrintDocument(). Сделай адаптер между принтером и редактором.

5. Нужно подключить старую систему уведомлений к новому модулю, не меняя старый код. Реализуй адаптер, который переведёт старый интерфейс в новый.


## Pattern.Builder задачи

1. Нужно реализовать создание пиццы с разными ингредиентами: тесто, соус, сыр, мясо, грибы. Сделай построение объекта Pizza через Builder.

2. Нужно собирать профиль пользователя: имя, email, возраст, адрес, номер телефона. Сделай Builder, чтобы не использовать огромный конструктор.

3. Нужно создавать конфигурацию компьютера: процессор, видеокарта, оперативная память, SSD. Реализуй Builder для сборки ПК по частям.

4. Нужно формировать заказ в интернет-магазине: товары, доставка, скидка, комментарий. Сделай Builder для удобной сборки заказа.

5. Нужно создавать отчёт, где могут быть заголовок, таблица, график, подпись. Реализуй Builder, чтобы поэтапно собирать объект отчёта.


## Pattern.ChainOfResponsibility задачи

1. Нужно обрабатывать обращения в техподдержку: простые запросы решает оператор, сложные — эксперт, критические — менеджер. Реализуй цепочку обработчиков.

2. Нужно проверять входящий запрос в веб-приложении: сначала авторизация, потом валидация, потом логирование. Сделай цепочку обработки запроса.

3. Нужно согласовывать заявку на отпуск: сначала руководитель, потом HR, потом директор. Реализуй передачу заявки по цепочке.

4. Нужно проверять платеж: сначала баланс, потом лимит, потом антифрод. Сделай цепочку обработчиков для одной операции.

5. Нужно фильтровать сообщение в чате: сначала проверка на спам, потом на запрещённые слова, потом на длину текста. Реализуй цепочку проверок.


## Pattern.Decorator задачи

2. Нужно добавить шифрование сообщения перед отправкой, не меняя код существующего уведомителя. Сделай декоратор шифрования.

3. Нужно считать время выполнения отправки уведомления. Реализуй декоратор, который замеряет время до и после вызова.

4. Нужно добавлять подпись к письму автоматически. Сделай декоратор, который дополняет текст сообщения перед отправкой.

5. Нужно комбинировать несколько дополнительных функций: логирование, шифрование, подпись. Реализуй цепочку декораторов поверх одного уведомителя.

5. Нужно создавать разные подключения к хранилищу: локальный диск, облако, FTP. Реализуй фабричный метод для выбора нужного подключения.

5. Нужно реализовать обход меню ресторана. Создай коллекцию блюд и отдельный итератор для её просмотра.


## Pattern.Observer задачи

3. Нужно отправлять обновление курса валют нескольким клиентам. Сделай объект-источник и подписчиков-наблюдателей.

4. Нужно уведомлять сотрудников о выходе новой задачи. Реализуй подписку и отписку от уведомлений.

5. Нужно оповещать клиентов об изменении статуса заказа. Создай наблюдаемого издателя и разные типы подписчиков.
